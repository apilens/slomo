**slomo** records everything your Python application does — every function call, SQL query, HTTP request, log record, and crash — into a local flight recorder. When something breaks, you don't reproduce the bug; you replay it.

```python
from slomo import enable

enable()
```

That's the entire integration. No dashboard, no browser, no Docker, no external database, no account, no telemetry. Everything is recorded locally under `.slomo/` and explored entirely from your terminal.

## Why a flight recorder?

Aircraft don't rely on pilots reproducing an incident — the black box already captured it. slomo brings the same model to Python debugging:

## Design guarantees

slomo is built to run in the background of a real application without you thinking about it:

- **Fast** — `enable()` completes in under 5 ms (enforced by a test). Events go through one lock-free queue put; a background thread batches writes. The recorder's hot path imports zero CLI dependencies.
- **Crash-safe** — append-only JSONL with fsync on the crash path and a tolerant reader. A `kill -9` loses at most the final partial line.
- **Never breaks your app** — every hook callback is exception-isolated, storage failures are swallowed, and backpressure drops low-severity events rather than blocking your code.
- **Private by default** — secrets are [redacted at capture time](/recording/redaction); they never touch disk.
- **Local-first** — your data stays in `.slomo/` in your project. JSONL timelines are the source of truth; the SQLite issue index is a rebuildable cache.
